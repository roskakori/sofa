Sofa Library

The purpose of Sofa Library is to allow accessing Amiga specific features using Eiffel. It consists of some Eiffel classes, a static link library sofa.lib and some additional tools.

Using the Sofa Library

To use the Eiffel classes, you have to extend the path for the compile command. This can be done by creating a file loadpath.se in the same directory where you start the compiler from (if such a file does not exist yet). Then, add the line (without leading or trailing blanks)

sofa:library/sofa/library/

Next, you have to tell the linker to link with sofa:lib/sofa.lib. Generally, you can do this by adding the option -l sofa:lib/sofa.lib when calling the Eiffel compiler using compile. The -l options will simply be passed on to the C compiler.

The only exception is SAS/c because it uses a different schema for command line options (based on ReadArgs() instead of the silly -something). Instead, create the file SCOPTIONS (in the same directory as loadpath.se) and add the line

LIBRARY=sofa:lib/sofa.lib

to it.

If you find this a bit confusing, just look at one of the examples coming with Sofa.

Interesting Classes

  • SIMPLE_REXX_CLIENT to send Rexx commands to some host
  • SIMPLE_REXX_SERVER for simple Rexx servers without GUI (based on SimpleRexx)
  • SHARED_AMIGA_DOS to access via a shared symbol Amiga_dos some DOS functions and DOS error reporting (IoErr(), Fault() etc)
  • SOFA_ARGUMENT_PARSER for simple command line argument parsing (using on ReadArgs())
  • SOFA_PATTERN to compare text with AmigaDOS pattern (based on ParsePattern() and MatchPattern())

Tools

  • make_sofa_version - make Eiffel class containing version information
  • readarghh - make command ling argument wrapper class

The following tools are merely interesting for people who want to contribute to Sofa or interface to C code:

  • make_export_header - extract exported prototypes from C source codes
  • make_rename_se - make rename.se file for Eiffel classes with too long names stored in a ZIP archive

Examples

You can find the source code for the sofa tools in sofa:library/sofa/example/ folder.
